**Dark mode wiring in `src/app/_layout.tsx`:**

The app uses NativeWind with `darkMode: 'class'` in tailwind.config.js. Since the existing codebase uses StyleSheet.create with `colors.X.light` everywhere (not Tailwind className dark: variants), the dark mode toggle needs to work with the imperative colors object. The simplest approach that works with the existing code:

1. Import `useSettingsStore` from `@/stores/settings`
2. Import `useColorScheme` from `react-native` (the native hook, not NativeWind)
3. In `RootLayoutInner`, read `themeOverride` from `useSettingsStore()`
4. Read `systemColorScheme` from `useColorScheme()` (returns 'light' | 'dark' | null)
5. Compute `effectiveTheme`: if themeOverride === 'system', use systemColorScheme ?? 'light'; otherwise use themeOverride
6. Pass effectiveTheme as a prop or via context. Since the existing code uses `colors.X.light` hardcoded throughout, the most pragmatic Phase 1 fix is to:
   - Import `Appearance` from `react-native`
   - Call `Appearance.setColorScheme(effectiveTheme)` inside a useEffect that depends on effectiveTheme. This sets the system-wide appearance for the RN app, which NativeWind's class-based dark mode will pick up.
   - This is a single useEffect addition, not a rewrite of every screen.

Add this code inside `RootLayoutInner` function body, after the existing useEffect for auth:
```typescript
const { themeOverride } = useSettingsStore();
const systemScheme = useColorScheme();

useEffect(() => {
  if (themeOverride === 'system') {
    Appearance.setColorScheme(null); // follow system
  } else {
    Appearance.setColorScheme(themeOverride);
  }
}, [themeOverride]);
```

Import `Appearance, useColorScheme` from 'react-native' (add to existing react-native import). Import `useSettingsStore` from '@/stores/settings'.

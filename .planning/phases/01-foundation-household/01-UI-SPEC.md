## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, avatar badge positioning, inline tag padding |
| sm | 8px | Button-to-label gap, list item internal padding |
| md | 16px | Default card padding, screen horizontal margin, input field padding |
| lg | 24px | Section heading bottom margin, card vertical gap |
| xl | 32px | Top-of-screen safe area padding, onboarding card padding |
| 2xl | 48px | Inter-section spacing on onboarding tour cards |
| 3xl | 64px | Bottom tab bar height + safe area composite |

Exceptions:
- Touch targets (buttons, tab icons, list items): minimum 44px height per iOS HIG and Android Material guidance — this overrides the token scale for interactive element minimum height only.
- Bottom safe area inset: device-native value via `react-native-safe-area-context`; do not hard-code.

## Typography

All sizes use React Native `fontSize`. React Native does not use CSS `line-height` in the same way — `lineHeight` must be absolute pixels. Computed as `fontSize × lineHeightMultiplier`.

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 16px | 400 (regular) | 24px (1.5×) |
| Label | 14px | 600 (semibold) | 20px (1.43×) |
| Heading | 20px | 600 (semibold) | 26px (1.3×) |
| Display | 28px | 600 (semibold) | 34px (1.21×) |

**Rules:**
- Only 2 font weights are used: 400 (regular) and 600 (semibold). Do not introduce 700 or 500.
- Body (16px/400) is the default for all paragraph, description, and list item text.
- Label (14px/600) is used for button text, tab labels, form labels, badge text, and chip text.
- Heading (20px/600) is used for screen titles, section headers, card titles, and modal titles.
- Display (28px/600) is used only on full-screen onboarding tour cards and the welcome screen headline.

**Source:** Standard typographic defaults for mobile; warm/approachable tone from CONTEXT.md `## App Identity & Brand`.

## Color

Warm and friendly palette. Dark mode ships from launch (CONTEXT.md). All color pairs listed as light/dark.

| Role | Light Value | Dark Value | Usage |
|------|-------------|------------|-------|
| Dominant (60%) | `#FFF8F0` (warm off-white) | `#1A1612` (warm near-black) | Screen background, page surface |
| Secondary (30%) | `#F5EDE0` (warm sand) | `#2C2420` (dark warm brown) | Cards, bottom sheet backgrounds, side drawer, tab bar background, input field fill |
| Accent (10%) | `#F97316` (warm amber-orange) | `#FB923C` (lighter amber-orange) | See "Accent reserved for" list below |
| Destructive | `#DC2626` (red) | `#EF4444` (lighter red) | Destructive actions only — leave household, remove member, delete account |
| Success | `#16A34A` (green) | `#22C55E` (lighter green) | Join confirmation, invite accepted, session established |
| Text primary | `#1A1612` (warm near-black) | `#FAF5F0` (warm near-white) | All body and heading text |
| Text secondary | `#78716C` (warm gray) | `#A8A29E` (lighter warm gray) | Captions, placeholder text, secondary metadata |
| Border | `#E7D9C8` (warm beige) | `#3D3530` (dark warm border) | Card borders, dividers, input outlines |

**Accent reserved for:**
- Primary action buttons ("Create Household", "Send Invite", "Join Household", "Continue" CTA)
- Active bottom tab indicator
- Selected avatar border
- AI credit meter fill bar
- Hyperlinks within body text

Accent is NOT used for: secondary buttons, navigation headers, badges/chips (those use secondary background), or decorative elements.

**"Jolly" AI assistant color:** Use accent (`#F97316`) for the Jolly character name in text, and as the background of the Jolly avatar badge. This is a sub-use of the accent budget.

**Source:** CONTEXT.md `## App Identity & Brand` — "warm & friendly", "rounded shapes, warm palette". Specific hex values: Claude's discretion within that direction.

## Interaction Patterns

| Pattern | Contract |
|---------|----------|
| Screen transitions | Expo Router slide-from-right for push navigation; fade for tab switches. No custom transitions in Phase 1. |
| Button press feedback | Scale to 0.96 on press-in, back to 1.0 on release. Duration: 100ms. Use `react-native-reanimated`. |
| Loading states | Full-screen spinner overlay (semi-transparent dominant background + accent spinner) for auth operations. Inline skeleton loaders for member list. |
| Error display | Inline below the triggering field (form errors). Toast-style bottom banner for network/API errors. Duration: 4 seconds auto-dismiss. |
| Bottom sheet | Drag to dismiss enabled. Backdrop opacity: 0.5 dominant color. Snap points: 50%, 90%. |
| Haptics | Light impact on button press. Notification feedback on successful join. Error feedback on auth failure. Use `expo-haptics`. |
| Dark mode | `useColorScheme()` hook from NativeWind. Manual override stored in Zustand `settingsStore`. Switches without app restart. |
| Animations | Minimal per CONTEXT.md — only: onboarding card swipe, bottom sheet slide, button press scale. No decorative animations. |

| Onboarding tour card 3 | Heading: "Jolly keeps it fair" / Body: "Your AI housemate helps balance chores, suggest meals, and track who owes what — so nobody keeps score." |

### Copy Rules

- Tone: warm, direct, no jargon. Write as if a friendly person is speaking, not a product team.
- Use "household" not "group", "team", or "home" in product UI copy.
- Use "Jolly" (capitalized) when referring to the AI assistant.
- Confirmation dialogs for destructive actions: always offer a Cancel first, destructive action second.
- Error messages: always state the problem + what to do next. Never show raw error codes.
- i18n: all copy goes into `src/locales/en/common.json` from day one. No hardcoded strings.

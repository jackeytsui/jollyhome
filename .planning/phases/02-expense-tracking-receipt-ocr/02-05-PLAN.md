    Create `src/hooks/useRecurring.ts`:

    ```typescript
    import { useState, useEffect, useCallback } from 'react';
    import { supabase } from '@/lib/supabase';
    import { useHouseholdStore } from '@/stores/household';
    import type { RecurringExpenseTemplate, SplitType } from '@/types/expenses';
    ```

    Returns: `{ templates, loading, createTemplate, updateTemplate, pauseTemplate, resumeTemplate, skipNext, deleteTemplate, processOverdue }`

    `loadTemplates()`:
    - Query `recurring_expense_templates` WHERE household_id, ordered by next_due_date ASC
    - Set templates state

    `createTemplate(input)`:
    - Input: `{ description, amount_cents, category, split_type, split_config (JSONB of split member/amounts), frequency, day_of_month?, day_of_week?, custom_interval_days?, next_due_date }`
    - Insert into `recurring_expense_templates` with household_id and created_by from stores/auth
    - Reload templates

    `updateTemplate(id, updates)`:
    - Update template fields. Partial update.

    `pauseTemplate(id)`:
    - Update `is_paused = true`

    `resumeTemplate(id)`:
    - Update `is_paused = false`

    `skipNext(id)`:
    - Advance `next_due_date` to the next occurrence based on frequency without creating an expense. Calculate next date:
      - daily: +1 day
      - weekly: +7 days
      - biweekly: +14 days
      - monthly: same day_of_month next month (handle month-end)
      - custom: +custom_interval_days days

    `deleteTemplate(id)`:
    - Delete from table

    `processOverdue()`:
    - Query templates WHERE `next_due_date <= today AND is_paused = false AND household_id = activeHouseholdId`
    - For each overdue template: call `supabase.rpc('create_recurring_expense_instance', { p_template_id: template.id })`
    - The RPC function atomically creates the expense and advances next_due_date
    - After processing, reload templates and trigger expense reload

    `useEffect` on mount: call `processOverdue()` when householdId is truthy. This is the client-side trigger — recurring expenses auto-create when the app opens.
  </action>

## Spacing Scale

Declared values (multiples of 4 only):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, tag padding, inline micro-spacing |
| sm | 8px | Stack gap between related items, separator margins |
| md | 16px | Card padding (established: Card.tsx uses 16), input padding, section padding |
| lg | 24px | Section heading gap, major element separation |
| xl | 32px | Screen-level vertical rhythm, between major sections |
| 2xl | 48px | Camera viewfinder margin, full-screen modal insets |
| 3xl | 64px | Bottom safe-area clearance for floating buttons |

Exceptions:
- Touch targets: minimum 44px height on all interactive elements (enforced in Button.tsx — `minHeight: 44`)
- Balance summary card collapse: 56px collapsed height
- Receipt framing guide: full-screen minus 48px top/bottom inset
- Item classification tag tap target: minimum 44x44px touch area, visual pill can be smaller

**Source:** Existing Button.tsx size styles + Card.tsx padding + 8-point scale default

|------|-------------|------------|-------|
| Dominant (60%) | `#FFF8F0` | `#1A1612` | Screen background, input background, camera overlay tint |

| Destructive | `#DC2626` | `#EF4444` | Delete expense confirmation, dispute badge, negative balance amounts |
| Success | `#16A34A` | `#22C55E` | Positive balance (owed money to you), settled debt indicator, shared item tags |
| Sandbox warning | `#CA8A04` | `#EAB308` | "Personal" item classification tags, partial settlement indicator |

| Text secondary | `#78716C` | `#A8A29E` | Dates, metadata, placeholders, captions |

**Accent (`#F97316`) reserved for:**
- Primary CTA buttons ("Add Expense", "Save", "Confirm", "Settle Up")
- Scan receipt FAB / camera trigger button
- Focused input border state
- Expense amount in list (money owed by you is NOT accent — use destructive)
- Jolly NL input field focus ring
- Category chip selected state
- Navigation active tab indicator

**Accent is NOT used for:**
- Balance amounts (use success green / destructive red)
- Item classification tags (use success green for Shared, sandbox amber for Personal)
- Disputed expense badges (use destructive)
- Settled state indicators (use success green)

**Source:** `src/constants/theme.ts` (all values) + 02-CONTEXT.md (item classification color-coded tags: green Shared / orange Personal)

## Component Inventory

New components required for Phase 2 (follow StyleSheet.create pattern):

### Expense Entry
- `JollyNLInput` — text field with "Tell Jolly..." placeholder, Jolly icon left, parsed-result callback. Guessed fields highlighted with 2px accent border + small "Jolly filled" chip above field.
- `QuickAddCard` — expandable Card with amount/description/split fields collapsed. "More details" chevron reveals category, notes, date, tax/tip, privacy toggle.
- `SplitTypeSelector` — horizontal scroll of chips: Equal / Percentages / Exact / Shares / [preset names]. Tapping opens a bottom sheet with member rows for input.
- `CategoryChipSuggestion` — inline pill below description input showing Jolly's category guess. Tappable to open full category picker list.
- `MemberSplitRow` — one row per member: avatar + name + amount/percentage input + toggle checkbox.

### Balance & Settlement
- `BalanceSummaryCard` — collapsible card. Collapsed: net balance headline. Expanded: per-member row (avatar, name, net amount in green/red). Tappable rows navigate to debt detail.
- `DebtDetailSheet` — bottom sheet: simplified debt chain, full/partial settle toggle, settle button, settlement history list below divider.
- `PaymentAppLinks` — horizontally scrollable chips for Venmo / Cash App / PayPal / Zelle. Member's preferred app rendered first with accent border. Each chip deep-links with amount pre-filled.
- `SettlementHistoryRow` — date, amount, method, running balance indicator.

### Expense List
- `ExpenseCard` — pressable Card: category icon + description + amount (right-aligned, accent) + date + payer avatar. Tap opens ExpenseDetailSheet. Long-press reveals context menu (edit/delete/dispute).
- `FilterBar` — horizontal scroll of filter chips: date range, category, member, amount range. Active filter chips filled with accent bg.
- `ExpenseDetailSheet` — full-screen modal with two tabs: "Details" (split breakdown, receipt thumbnail, category, privacy badge) and "History" (change log rows with timestamp + actor + diff).
- `DisputeBadge` — small orange/amber pill with flag icon. Renders on ExpenseCard when dispute is open.
- `ChangeHistoryRow` — actor avatar + action text + timestamp.

### Receipt Scanning
- `ReceiptCameraView` — full-screen camera with corner-bracket framing guide (4 accent-colored corner marks, no full border). Auto-capture shimmer animation when receipt detected. Manual shutter button (64px circle, white bg, accent border) at bottom center. Cancel button bottom-right: label "Discard Scan", accessibilityLabel "Discard Scan".
- `ReceiptPageStack` — multi-page indicator (dot row) + "Add another page" button for long receipts.
- `ReceiptReviewCard` — Card with store name (heading), date, scrollable line items list, tax row, tip row, total row. Each field editable inline (tap to edit).
- `ItemClassificationTag` — pill on each receipt line item: green "Shared" or amber "Personal: [Name]". Tap cycles through: Shared -> Personal:Me -> Personal:[Member...] -> back to Shared. Haptic feedback on toggle.
- `SplitSummaryPreview` — below item classification list: per-person total (shared portion + personal items + proportional tax). One "Confirm & Save" CTA.

### Recurring Expenses
- `RecurringExpenseRow` — template list item: description + amount + schedule label + next-due date. Swipe actions: skip / pause / edit.
- `RecurrenceSchedulePicker` — bottom sheet with frequency options and day selector.

### States
- `ExpenseSkeletonCard` — loading placeholder with shimmer animation matching ExpenseCard dimensions.
- `BalanceSkeletonCard` — loading placeholder for BalanceSummaryCard collapsed height.
- `OfflineBanner` — thin amber banner at top: "You're offline. New expenses will sync when reconnected." Uses sandbox color.
- `JollyParsingIndicator` — Jolly icon with typing dots animation shown in JollyNLInput while AI parses.

## Interaction Contracts

### Haptics (expo-haptics)
- `ImpactFeedbackStyle.Light` — every button press (consistent with Phase 1 Button.tsx)
- `ImpactFeedbackStyle.Medium` — ItemClassificationTag toggle, settle debt confirmation
- `ImpactFeedbackStyle.Heavy` — receipt auto-capture trigger
- `NotificationFeedbackType.Success` — expense saved successfully, settlement confirmed
- `NotificationFeedbackType.Error` — NL parse failed, network error

### Animation (react-native-reanimated)
- Button press scale: 0.96 in 100ms, 1.0 out 100ms (consistent with Phase 1)
- Card press scale: 0.98 in 100ms, 1.0 out 100ms (consistent with Phase 1)
- BalanceSummaryCard collapse/expand: height interpolation 200ms ease-out
- JollyParsingIndicator: looping dot fade cycle 400ms each
- OfflineBanner: slide-in from top 250ms ease-out, slide-out 200ms ease-in
- Skeleton shimmer: translateX loop from -width to +width 1.2s, repeat

### Loading States
- Initial tab load: BalanceSkeletonCard + 3x ExpenseSkeletonCard rows
- Jolly NL parsing: JollyParsingIndicator in input field, input disabled
- Receipt OCR processing: full-screen loading overlay with Jolly icon + "Reading your receipt..." text + progress-style shimmer on preview image
- Settlement marking: spinner replaces "Settle Up" button label, 300ms delay before optimistic UI update

### Offline Behavior
- Show OfflineBanner at top of screen
- Cached expenses render read-only (no edit/delete swipe actions)
- Quick-add form stays enabled but "Add Expense" button label changes to "Save for later"
- Queued expenses show amber dot badge on tab icon

## Debt Simplification UX (Claude's Discretion area)

**Display model:** Show simplified debts by default. Toggle "Show original" chevron on DebtDetailSheet reveals the un-simplified chain with a brief explanation: "3 debts simplified to 1 transfer."

**Visual treatment:**
- Simplified debt: primary card row, full opacity
- Original chain debts (in expanded view): secondary row, 60% opacity, smaller font (Label: 14px / 400 — maps to Caption alias)
- Simplification savings indicator: small chip "Saved 2 transfers" in accent bg at bottom of expanded section
